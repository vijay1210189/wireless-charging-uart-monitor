📘 Detailed Project Description (Paragraph Format)
In this project, a complete wireless charging monitoring and diagnostics system was designed and implemented by combining embedded system simulation with a web-based visualization dashboard. The primary objective of the project was to simulate how a wireless charging system communicates through UART, process that data at a protocol level, and display meaningful engineering parameters in a structured graphical interface similar to industrial tools like LabVIEW. The project begins at the embedded layer, where a UART response frame is simulated using C programming. The simulated frame mimics actual hardware communication by encoding multiple system parameters such as battery voltage, rail voltage, current, power values, charger phase, and error flags into specific byte positions within a data packet.
A key part of the implementation involves byte offset mapping, where each parameter in the system is mapped to a particular index in the UART response array. This reflects real-world embedded communication protocols where raw byte streams must be interpreted to extract meaningful information. For example, certain byte positions store the selected coil number, others represent system status, and a group of bytes collectively represent error flags in a 32-bit format. By decoding these byte positions correctly, the system can reconstruct the internal state of the wireless charging system. This approach demonstrates a strong understanding of low-level communication and data parsing techniques used in embedded systems.
The project also includes an implementation of a charger state machine, which models the progression of a wireless charger through different phases such as DEFAULT, PING, CONFIGURATION, NEGOTIATION, and POWER TRANSFER. Each phase represents a stage in the charging process, and the system tracks transitions between these states. This state machine logic reflects how actual wireless charging firmware operates and ensures that the system behaves realistically. By displaying the current phase in the dashboard, the user can easily understand the system’s operational state at any given time.
In addition to basic parameter monitoring, a detailed diagnostics module was developed to detect faults and abnormal conditions. The system uses a 32-bit error flag variable where each bit corresponds to a specific fault, such as Foreign Object Detection (FOD) error, communication timeout, or excessive power loss. These error flags are decoded and represented visually in the dashboard using LED-style indicators, where red denotes an active fault and green indicates normal operation. This visual representation closely mimics real embedded debugging tools, making the interface intuitive for engineers.
A major part of the project is dedicated to Foreign Object Detection (FOD), which is a critical safety feature in wireless charging systems. The project includes both Open FOD status and Real FOD verification calculations. Parameters such as Q factor, Q margin, delta, coefficients, power thresholds, and calibration values are simulated and displayed. This demonstrates an understanding of the efficiency and safety mechanisms involved in wireless power transfer systems. The inclusion of these parameters adds depth to the project by going beyond simple monitoring and incorporating system-level control and verification logic.
To make the system user-friendly and visually interpretable, a web-based dashboard was developed using HTML and CSS. The dashboard is structured into multiple panels, each dedicated to specific aspects of the system. The top panel provides UART configuration controls such as port and baud rate, along with action buttons like connect, monitor, pause, and stop. The main section is divided into a grid layout where the left side displays charging status parameters and the right side displays error flags. Additional panels show FOD-related information and verification results. At the bottom, a structured table displays UART response data with clear mapping between byte indices and their corresponding meanings.
The dashboard design focuses on compactness and clarity, following a LabVIEW-like style with minimal wasted space and well-aligned elements. Parameters are presented in labeled boxes, and color coding is used to highlight important values such as power loss or error conditions. LED indicators provide quick visual feedback for system status, while the UART table allows engineers to trace how raw data maps to processed values. This combination of low-level and high-level visualization makes the system both informative and easy to use.
The entire project was organized into a structured repository with separate directories for source code, documentation, and dashboard files. Detailed documentation files were included to explain byte mappings, charger phases, and FOD concepts, ensuring that the project is easy to understand and maintain. Version control was managed using Git, and the project was ultimately deployed using GitHub Pages, making the dashboard accessible as a live web application.
Overall, this project demonstrates a complete workflow from embedded system design to user interface development. It combines concepts such as UART communication, state machines, fault diagnostics, and web-based visualization into a cohesive system. The result is a professional-grade monitoring tool that reflects real-world engineering practices used in embedded and power electronics domains. This project highlights strong skills in embedded C programming, protocol handling, debugging, system design, and frontend development, making it a valuable addition to any technical portfolio.

📘 Detailed Explanation of All Parameters in the Project
The project involves monitoring and analyzing a wide range of electrical, control, and diagnostic parameters that are typically present in a wireless charging system. These parameters are extracted from a simulated UART response packet and represent the internal state of the charging system at any given moment. Each parameter has a specific role in understanding system performance, safety, and efficiency.
At the core of the system are the fundamental electrical parameters such as battery voltage, rail voltage, and current. The battery voltage represents the voltage level of the device being charged and is a key indicator of charging progress and battery health. The rail voltage, on the other hand, reflects the internal power distribution level within the system and is essential for ensuring proper power regulation. The current parameter indicates how much electrical current is flowing during charging, which directly affects charging speed and efficiency. Together, these parameters provide insight into how effectively power is being supplied and utilized.
In addition to these, the system computes and displays transmit power (TX power) and receive power (RX power). TX power represents the power transmitted by the charging pad, while RX power represents the power received by the device. The difference between these two values gives the power loss, which is an important metric in wireless charging systems. Power loss helps identify inefficiencies in energy transfer and can indicate potential issues such as misalignment or foreign object interference. Highlighting power loss in red within the dashboard makes it easier to quickly identify abnormal conditions.
Another important group of parameters relates to the operational control of the system. The charger phase parameter represents the current state of the wireless charging process. The system transitions through several phases such as DEFAULT, PING, CONFIGURATION, NEGOTIATION, and POWER TRANSFER. This phase information is crucial because it tells the user exactly what stage of the charging process the system is in. For example, if the system is stuck in negotiation, it may indicate a communication or compatibility issue.
The project also includes parameters such as signal strength and operating frequency, which are related to the communication and energy transfer mechanism. Signal strength indicates how effectively the transmitter and receiver are communicating, while frequency represents the oscillation rate used for wireless power transfer. These parameters help diagnose issues related to electromagnetic coupling and system tuning.
Temperature monitoring is another critical aspect of the system. The project includes multiple temperature parameters such as Temp1, Temp2, and Temp3, which simulate readings from different thermal sensors. These values are important for ensuring the system operates within safe limits. Excessive temperature can damage components or indicate inefficiencies, so monitoring temperature is essential for both safety and reliability.
Control-related parameters such as control error and control interval are also included to reflect the internal feedback mechanisms of the system. Control error indicates the difference between expected and actual performance, while control interval represents the timing of control updates. These parameters are useful for understanding how well the system is regulating power and maintaining stability.
A major component of the project involves error flags, which are stored as a 32-bit value. Each bit in this value corresponds to a specific fault condition, such as Foreign Object Detection (FOD), overcurrent, communication timeout, or excessive power loss. By decoding these bits, the system identifies which errors are currently active. These decoded errors are visually represented using LEDs in the dashboard, providing a quick and intuitive way to understand system faults.
The project also includes advanced parameters related to Foreign Object Detection (FOD), which is a critical safety mechanism in wireless charging. In the Open FOD status, parameters such as Q normal, Q margin, delta, and coefficients are used to evaluate the system’s efficiency and detect anomalies. These values help determine whether any unexpected object is interfering with the power transfer process. In the Real FOD verification section, additional parameters such as PTX (transmit power), PRX (receive power), threshold, margin, and calibration values are used to validate system behavior under real operating conditions. Parameters like alpha and gamma are included to represent calibration factors used in advanced algorithms.



Finally, all these parameters are linked back to the UART response structure, where each value is extracted from a specific byte index. This linkage ensures that the dashboard is not just displaying arbitrary data, but is actually representing a decoded communication protocol. This connection between raw data and high-level visualization is one of the key strengths of the project.
Overall, the parameters in this project collectively provide a comprehensive view of the wireless charging system, covering electrical performance, operational state, safety diagnostics, and communication integrity. By integrating all these parameters into a single dashboard, the project successfully demonstrates how complex embedded data can be transformed into an intuitive and informative user interface.
